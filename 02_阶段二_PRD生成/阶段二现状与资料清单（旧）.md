## 二、技能概要

### 五步工作流程

```
步骤一：资料输入与解析
  ├── 1.1 接收参考资料（文件上传/文字描述）
  ├── 1.2 资料类型识别与提炼（7个维度）
  └── 1.3 信息确认

步骤二：PRD 大纲生成
  ├── 2.1 自动生成大纲（8章+3附录标准结构）
  ├── 2.2 大纲弹性调整规则
  └── 2.3 确认产品正式全称

步骤三：分章节顺序输出（1→2→3→...→8→附录）
  └── 每章写初稿 → 自检 → 提交确认

步骤四：迭代修订
  ├── L1/L2/L3 三级变更级别
  ├── 5轮迭代预警
  └── 双条件退出（用户确认 + 全流程自检通过）

步骤五：终稿输出与归档
```

### PRD 标准章节结构（8章+3附录）

| 章节 | 内容 | 关键产出物 |
|:----:|------|-----------|
| 第1章 | 产品概述 | 产品定位/用户画像/使用场景/约束/术语 |
| 第2章 | 功能需求总览 | 功能基准矩阵/模块决策表/功能清单/权限矩阵 |
| 第3章 | 系统核心业务流程 | ASCII流程图/业务节点说明/系统间数据流转 |
| 第4章 | 功能架构设计 | 功能模块结构图/模块定位/依赖关系 |
| 第5章 | 用户体验与页面结构 | 用户故事链/一级导航/跳转关系图/页面清单 |
| 第6章 | 页面详细设计 ⭐ | 逐页ASCII布局/元素表/交互行为表/异常处理/弹窗清单 |
| 第7章 | 非功能需求 | 性能/安全/集成/可用性 |
| 第8章 | 核心用例规约 | 用例ID/前置条件/操作步骤/预期结果 |
| 附录A | 迭代变更记录 | L1/L2/L3 变更日志 |
| 附录B | 版本快照历史 | 各阶段快照与版本对应关系 |
| 附录C | 全流程自检报告 | 逐章检查清单 |
