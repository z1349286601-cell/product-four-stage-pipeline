## 两段式结构

产品设计会话包含两个阶段，分两段执行：

### 前半段：Stage 2 — 业务功能分析

| 步骤 | 内容 | 写入位置 |
|------|------|---------|
| 3.2.1 产品定位 | 产品定位陈述 + 用户画像 + 核心场景 + 约束 | PRD.md 第 1 章 |
| 3.2.2 功能矩阵 | 竞品功能对比 → 模块决策（保留/增强/新增/删除） | PRD.md 第 2 章 |
| 3.2.3 业务流程 | Mermaid 流程图 + 节点说明 + 数据流转 + 术语 | PRD.md 第 3-4 章 |

**本阶段无独立文件产出**。所有确认后的内容通过对话暂存，在 Stage 3 统一写入 PRD.md。

### 后半段：Stage 3 — 线框图与 PRD

| 步骤 | 内容 | 产物 |
|------|------|------|
| 4.2 页面清单 | 编号/名称/路由/类型/模块/功能点 → 用户二次确认 | `页面清单.yaml` |
| 4.3 DESIGN.md | 确认设计基底 | — |
| 4.4 逐页线框图 | 24 列网格 + Excalidraw，每页标注交互/异常/弹窗 | `线框图/PC-XX-名称.excalidraw` |
| 4.5 PRD 生成 | 8 章完整需规（第 6 章引用线框图路径） | `PRD.md` |
| 4.6 同步校验 | 线框图 ↔ PRD 6 项一致性检查 | `_同步校验记录.md` |

## PRD 章节目录（v3.0）

```
第1章 产品概述        → 定位/用户/场景/约束/术语
第2章 功能需求总览      → 功能基准矩阵/模块决策/功能清单/权限矩阵
第3章 功能架构设计      → 模块结构/分工/依赖关系
第4章 系统核心业务流程   → Mermaid 流程图/节点说明/数据流转
第5章 用户体验与页面结构 → 导航结构/跳转关系/页面清单
第6章 页面详细设计      → 每页：线框图引用/元素表/交互表/异常表/弹窗清单
第7章 非功能需求       → 性能/安全/集成/可用性
第8章 核心业务闭环验证   → 3-5 条端到端流程闭环
附录                     → 变更记录/版本快照/自检报告
```

## 关键约束

- 第 6 章每个页面必须写 `**线框图**：02-产品设计/线框图/PC-XX-名称.excalidraw`
- 禁止 ASCII 布局描述（线框图是布局权威源）
- 交互行为表和异常处理表从线框图标注中提取，确保一致
- 页面清单确认后不再轻易改动（影响原型范围）
