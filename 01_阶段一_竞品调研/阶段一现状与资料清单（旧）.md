## 二、技能概要

### 三阶段结构

```
第一阶段：需求澄清（必做）
  ├── 1.1 识别输入信息（A/B/C/D 四种权重等级）
  ├── 1.2 结构化澄清对话
  └── 1.3 调研目标确认单

第二阶段：七步调研法
  ├── 第一步：确定调研对象（本行业标杆 / 跨行业类似产品）
  ├── 第二步：从搜索开始（中文/英文搜索策略）
  ├── 第三步：浓缩关键词（从头部玩家提炼术语）
  ├── 第四步：咨询和学术领域找答案（咨询报告/书籍/论文）
  ├── 第五步：扩展阅读清单（内容账号/书籍推荐）
  ├── 第六步：真实体验demo（视频/截图/试用）
  └── 第七步：从商业角度看产品（产品角度 vs 商业角度）

第三阶段：输出物模板
  └── 调研报告.md（单文件，含全部内容）
```

### 调研报告模板章节

1. 调研目标确认单
2. 行业概况（市场趋势/产业链/主流玩家）
3. 竞品分析（产品定位对比）
4. 详细功能清单对比表
5. UML图（PlantUML：用例图/活动图/时序图/状态机图）
6. 行业适配分析
7. 竞品对比总结（优势/短板/差异化机会）
8. 功能优先级建议（P0/P1/P2）
9. 参考资料清单
10. 附录（术语表/调研方法记录/待深入研究项）

## 四、衔接问题诊断

### 阶段一 → 阶段二的传递问题

**当前传递方式**：调研报告.md → 作为 PRD 生成器的「参考资料」输入

**存在痛点**：
1. 功能清单对比表在调研报告第4章，PRD生成器需要人工识别并填入「2.1功能基准矩阵」
2. 竞品优先级建议在调研报告第8章（P0/P1/P2），与PRD的「2.2模块决策表」需要对齐但无自动映射
3. UML用例图/活动图可直接复用到PRD第3章（业务流程），但当前靠人工搬运
4. 调研结论中的「差异化机会」如何转化为PRD中的功能设计，缺少方法论指引
