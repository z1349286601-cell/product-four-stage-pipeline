# PRD 生成技能

基于用户提供的多源参考资料，逐步引导生成完整的产品需求规格说明书（PRD）。所有产出物集中在一个 `.md` 文档中，不拆分独立文件。

## 核心原则

- **大纲驱动**：从资料中自动提炼 PRD 大纲，用户确认后再按章节顺序分阶段输出
- **迭代闭环**：允许回溯修改大纲或已写章节，每次改动通盘检查各环节间的逻辑闭环与前后一致性
- **集中输出**：所有内容统一写入 PRD 主文档，零外部附件
- **迭代可追溯**：每次重大调整记录变更版本，支持回溯
- **统一设计规范**：页面详细设计（第6章）为固定流程章节，包含页面路径、线框图引用、元素说明、跳转关系、关联弹窗、交互行为、异常处理。布局权威源为线框图（Excalidraw），PRD 中不再重复 ASCII 描述。

---

## 完整工作流程

```
步骤一：资料输入与解析
    ↓
步骤二：PRD 大纲生成 → 用户确认/修订
    ↓
步骤三：分章节顺序输出 → 每章用户确认
    ↓
步骤四：迭代修订（允许回溯改大纲/章节，通盘一致性检查）
    ↓
步骤五：终稿输出与归档
```

## 步骤一：资料输入与解析

### 1.1 接收参考资料

引导用户提供参考资料，支持两种方式混合输入：

| 输入方式 | 说明 |
|---------|------|
| 文件上传 | 读取用户提供的 `.md` / `.txt` 文件（支持多文件） |
| 文字描述 | 用户在对话中直接输入的背景说明、需求描述等 |

### 1.2 资料类型识别与提炼

从用户提供的资料中识别并提炼以下关键信息：

| 信息维度 | 识别内容 |
|---------|---------|
| 产品定位 | 目标行业、客群层级、核心价值主张、产品名称（临时） |
| 竞品参考 | 竞品名称、竞品功能模块、可参照的设计模式 |
| 核心业务场景 | 场景名称、核心流程、主要用户角色 |
| 初步解决方案 | 已构想的方案思路、技术路线、关键能力方向 |
| 业务单据与数据流 | 核心业务单据、数据实体、系统间数据流转关系 |
| 系统核心能力 | 需要构建的核心功能/能力、AI 能力方向 |
| 项目约束 | 预算与部署、数据基础设施、系统集成、合规标准、物联网接入等 |

### 1.3 信息确认

- 若资料覆盖不全，通过对话引导用户补充关键信息
- 确认产品定位、使用场景、项目约束三个维度的核心信息

### 步骤一产出
- 资料提炼摘要（行业/客群/价值主张/场景/约束）
- 待补充信息清单（如有）

## 步骤二：PRD 大纲生成

### 2.1 自动生成大纲

基于步骤一提炼的信息，自动生成 PRD 大纲。大纲结构以标准模板为基础，根据资料内容弹性调整：

**标准章节结构**：

```markdown
第1章 产品概述
  1.1 产品定位
  1.2 目标用户画像
  1.3 核心使用场景
  1.4 约束与依赖
  1.5 核心名词与术语

第2章 功能需求总览
  2.1 功能基准矩阵（竞品对比）
  2.2 模块决策表（保留/增强/新增/删除）
  2.3 功能清单详版（分模块）
  2.4 用户角色与权限矩阵

第3章 功能架构设计
  3.1 功能模块结构图
  3.2 模块定位与分工说明
  3.3 模块业务依赖关系图

第4章 系统核心业务流程
  4.1 核心业务流程总览（含 Mermaid 流程图）
  4.2 关键业务节点说明
  4.3 系统间数据流转

第5章 用户体验与页面结构设计
  5.1 核心用户故事链（3-5条）
  5.2 一级导航结构
  5.3 页面跳转关系（Mermaid 图 + 关键跳转说明）
  5.4 页面清单（编号/名称/类型/承载功能/归属模块）

第6章 页面详细设计 🆕
  6.1 模块X — 功能概述
  6.2 页面 X-X：页面名称
    - 页面路径
    - 线框图引用（布局权威源，不重复 ASCII 描述）
    - 页面元素说明
    - 交互行为
    - 异常处理
    - 关联弹窗/子页面（含弹窗清单表格）

第7章 非功能需求
  7.1 性能需求（按模块分条）
  7.2 安全合规需求
  7.3 系统集成需求
  7.4 可用性与运维需求

第8章 核心业务闭环验证
  8.1 闭环说明（选取 3-5 条贯穿多页面的端到端流程）
  8.2 闭环详情（ID/涉及页面/操作步骤/预期结果，简要描述，不重复第6章已有内容）

附录A：迭代变更记录
附录B：版本快照历史
附录C：全流程自检报告
```

### 2.2 大纲弹性调整规则

- 若用户资料中某章节对应的信息不足，标注「待补充」而非删除章节
- 若用户明确不需要某章节，可跳过但保留章节占位
- 大纲中标注每个章节预计的产出物

### 2.3 确认产品正式全称

**执行时机**：大纲确认后、开始写第1章之前。

**必问问题**：请输入本次需求规格说明书对应的**产品正式全称**：_______

> 该名称将统一应用于：PRD 文档标题、所有章节标题、产出物命名、版本快照命名。

**若用户未给出**：使用临时名称生成，后续通过变更记录修正所有引用位置。

### 步骤二产出
- PRD 大纲（含章节结构、对应资料摘要、待补充标注）
- 产品正式全称确认记录

### 步骤二定稿条件
1. 用户明确确认「大纲确认，可进入下一阶段」
2. 产品正式全称已确认

## 步骤三：分章节顺序输出

### 3.0 分两段执行

步骤三分为两个阶段：

**前半段（对应 pipeline stage-2）**：第 1-3 章
- 产品概述、功能需求总览、功能架构
- 内容通过对话确认后，暂存于对话上下文，不创建独立文件
- 用户确认后进入后半段

**后半段（对应 pipeline stage-3）**：第 4-8 章 + 线框图
- 业务流程、页面结构、线框图（Excalidraw）、页面详细设计（第6章引用线框图）、非功能需求、业务闭环验证
- 产出完整 PRD.md（含全部 8 章）+ 页面清单.yaml + _同步校验记录.md

#### 3.0.1 线框图绘制规范（强制）

在进入第 5 章页面结构设计前，先输出线框图。以下规则为强制约束：

**格式**：
- 线框图**必须**是 `.excalidraw`（JSON 结构化矢量图），**严禁**用 `.md` ASCII 伪图替代

**图表区域**：
- ✅ 只画：容器框 + 标题 + 数据源标注 + 占位标记（如 `[ 图表区 · 占位 ]`）
- ❌ 禁止：用 line/rect/ellipse 等元素模拟数据点、柱状条、弧线、折线走向
- 原因：模拟数据图形绘制成本高且不准确，线框图阶段不需要确定具体数据可视化形式

**逐页确认**：
- 每页线框图完成后**必须停止**，等待用户确认后再画下一页
- **严禁**一次性批量输出多页线框图
- 即使页面数量多（如 8+ 页），也必须遵守逐页节奏

### 3.1 章节输出流程

按 1→2→3→...→8→附录 的顺序逐章输出。每章流程：

1. **写本章初稿**：基于资料 + 前文已确认章节，生成本章内容
2. **自检**：检查本章与前文的逻辑一致性、术语一致性、数据一致性
3. **提交用户确认**：用户确认后进入下一章

### 3.2 第6章页面详细设计规范（核心章节）

第6章以第5章输出的页面清单和线框图为输入，按模块分组对每个页面进行详细设计。**最小粒度到弹窗/子页面**。线框图是布局的权威源，PRD 中只引用不重复描述。

#### 页面详细设计模板（每个页面）：

```markdown
#### X.X.X 页面 X-X：页面名称

**页面路径**：侧边菜单 > XXX

**线框图**：`02-产品设计/线框图/PC-XX-名称.excalidraw`
（布局参见线框图，PRD 中不重复描述布局。线框图是布局的权威源。）

**页面元素说明**：
| 区域 | 元素 | 类型 | 说明 |
|------|------|------|------|
| XX区 | XX名称 | 组件类型 | 详细说明（含颜色、宽度、数据绑定等） |

**交互行为**（从线框图标注提取）：
| 触发动作 | 前置条件 | 系统响应 | 反馈方式 |
|---------|---------|---------|---------|

**异常处理**（从线框图标注提取）：
| 场景 | 系统行为 | 用户提示 |
|------|---------|---------|

**关联弹窗/子页面**：
| 序号 | 弹窗名称 | 触发入口 | 宽度 | 内容概要 |
|:----:|---------|---------|:---:|---------|
```

#### 第6章编排规范

- 按功能模块（对应第4章的 L1 模块）分组，每个模块下包含该模块的所有页面
- 每个模块开头写「功能概述 + 核心流程简述 + 子页面清单」
- 页面路径明确标注导航层级，弹窗标注触发入口
- **线框图引用**：每个页面写 `**线框图**：路径`，不重复 ASCII 布局。线框图是布局的权威源
- 交互行为表和异常处理表从线框图标注中提取，确保与线框图一致
- 表格列定义需标注 dataIndex 和宽度
- 状态标签需标注颜色色值

### 3.3 每章自检要点

| 检查项 | 说明 |
|--------|------|
| 术语一致性 | 本章术语是否与前文定义一致 |
| 数据一致性 | 数据实体、状态值是否与前文矛盾 |
| 模块引用一致性 | 模块编号/页面编号是否与前文对应 |
| 角色一致性 | 角色定义是否与第2章权限矩阵一致 |
| 产品名称一致性 | 文档内所有产品名称是否统一 |

### 步骤三产出
- 逐章产出的 PRD 内容

## 步骤四：迭代修订

### 4.1 迭代触发条件

- 用户对已确认的章节提出修改
- 用户要求修订大纲结构
- 用户补充新资料影响已写内容

### 4.2 迭代处理流程

1. **定位影响范围**：分析改动涉及哪些已写章节
2. **影响评估**：
   - 大纲层面：章节增删/结构调整 → 评估对已写内容的影响
   - 内容层面：单章修改 → 检查是否影响前后章节的逻辑闭环
3. **通盘一致性检查**：按附录 C 全流程自检清单逐条核验
4. **生成变更记录**：按 L1/L2/L3 级别记录
5. **提交用户确认修改**

### 4.3 变更级别定义

| 级别 | 范围 | 强制要求 |
|------|------|---------|
| L1 | 产品定位/大纲结构变更（影响全局） | 附带全流程影响评估 |
| L2 | 模块范围变更（影响局部模块） | 附带架构/页面影响评估 |
| L3 | 功能细节/页面元素变更（单点调整） | 附带局部影响说明 |

### 4.4 迭代预警

每轮迭代结束时检查：已迭代超过 5 轮时提醒用户「当前已进行5轮迭代，建议收敛锁定版本，后续以增量调整为主。」

### 4.5 迭代退出条件（双条件）

1. 用户明确确认「当前版本可定稿」
2. 全流程自检通过：无逻辑断点、无术语矛盾、无模块遗漏

## 步骤五：终稿输出与归档

### 5.1 生成需求快照

每次重大节点（大纲确认/全文定稿）生成版本快照：

```markdown
【需求快照 R1.0】
生成时间：YYYY-MM-DD HH:MM
产品正式全称：_______________
本次变更级别：-
变更内容：全文定稿
影响评估：
  - 影响模块：全部
  - 文档状态：已定稿
```

### 5.2 输出文件

- **文件命名**：`{产品正式全称}_产品需求规格说明书_V{版本号}.md`
- **保存路径**：当前工作空间目录

### 5.3 文档头部模板

```markdown
# 《产品正式全称》产品需求规格说明书

> 版本号：V1.0
> 生成日期：YYYY-MM-DD
> 生成工具：PRD 生成 Skill
> 对应需求快照：R1.0
> 文档状态：初稿/评审中/已定稿
> 确认人：___________
```

### 步骤五产出
- 完整 PRD 文档（单文件，含 8 章 + 3 附录）
- 版本快照记录

## 页面设计强制标准

| 标准 | 强制要求 |
|------|---------|
| 页面清单完整性 | 第5章必须输出完整的页面清单（编号/名称/类型/承载功能/归属模块） |
| 线框图格式强制 | 线框图**必须**是 `.excalidraw`（JSON 结构化矢量图），**严禁**用 `.md` ASCII 伪图替代 |
| 线框图完整性 | 每个页面必须有对应的 Excalidraw 线框图文件（24列网格系统），标注区域、数据绑定、交互、异常态 |
| 线框图图表规范 | 图表区域**不画具体数据图形**：只画容器框+标题+数据源标注+占位标记（如 `[ 图表区 · 占位 ]`），禁止用 line/rect/ellipse 模拟数据点/柱/弧线 |
| 线框图逐页确认 | 每页完成后**必须停止**等待用户确认，**严禁**一次性批量输出多页 |
| 页面详细设计完整性 | 第6章每个页面必须包含：线框图引用、元素说明表、交互行为表、异常处理表 |
| 线框图引用强制 | 第6章每个页面必须写 `**线框图**：路径`，禁止用 ASCII 布局替代 |
| PRD ↔ 线框图同步 | 交互行为表和异常处理表必须从线框图标注中提取，确保二者一致 |
| 弹窗覆盖完整性 | 第6章必须包含所有关联弹窗/子页面的清单与规格说明 |
| 一级菜单落地 | 每个一级菜单必须有落地页，不能仅作为导航容器 |
| 页面跳转覆盖率 | 100% 有业务关联元素支持跳转 |
| 孤立功能数量 | 0 个（所有功能必须嵌入页面或弹窗） |

## 输出规范

- **主文档格式**：单文件 `.md`，命名 `{产品正式全称}_产品需求规格说明书_V{版本号}.md`
- **附加产物**（与 PRD.md 同目录）：
  - `页面清单.yaml`：结构化页面索引，供原型开发阶段快速解析
  - `_同步校验记录.md`：PRD ↔ 线框图 一致性检查记录
- **产品名称一致性**：文档标题、所有章节标题、产出物命名、版本快照全部统一使用步骤二确认的正式全称
- **变更记录**：附录 A 记录所有 L1/L2/L3 变更，含日期、内容摘要、影响范围、确认人
- **版本快照**：附录 B 绑定各阶段快照与 PRD 版本的对应关系
