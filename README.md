## 核心发现：两套技术方案的分叉

```
路径A（传统）：Vue 3 + Ant Design Vue 4 → 独立工程 → 五级验收
路径B（make）：React 18 + Ant Design 5 + Tailwind → @axhub/make-client → 标注+验收脚本

关键冲突：两条路径技术栈完全不同，不可互通
```

---

## 最大变量：axhub make 引入的范式转变

make 不是简单的原型工具，它重新定义了「原型→文档」的关系：

```
传统：PRD（主）+ 原型（辅）→ 两者分叉，返工严重
make：原型 + 标注（主）+ 轻量文档（辅）→ 统一入口，标注即文档
```

这会影响整个四阶段流水线的设计逻辑。
