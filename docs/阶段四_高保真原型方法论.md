## 职责

在 `@axhub/make-client` 项目体系内，基于上游产物（PRD + 页面清单 + 线框图 + DESIGN.md）生成可交互、可直接用于客户演示的原型代码。

## 设计基底：DESIGN.md v3.1

### 核心决策

| 决策 | 说明 |
|------|------|
| **设计体系** | Ant Design Pro 风格（对齐 `preview.pro.ant.design`） |
| **依赖策略** | 零额外 Pro 组件依赖。ProCard 不兼容 antd 6，改用 StatCard/ChartCard 组件模板 |

## 模板启动机制

原型开发不再从零手写，而是从两个模板启动：

```
Step 1: 复制 make 工程模板（一次性，由用户手动操作）
  cp -r make-project-template/ D:/AI/workspace/axhub_workspace/<新项目>/
  cd <新项目> && npm install

Step 2: 从 _template 创建原型骨架（AI 执行）
  cp -r make-project-template/src/prototypes/_template/ src/prototypes/<原型名>/

Step 3: 定制 TODO 占位
  替换菜单配置、路由、页面映射、Logo、数据导入路径

Step 4: 基于 Dashboard.tsx 模式开发各页面
```

## 执行流程

```
步骤一：环境准备与上游读取
  ├─ 1.1 读流水线状态 → 确认 stage-2+3 done
  ├─ 1.2 获取 make 项目路径 → 路径校验 Gate（失败必须向用户确认）
  ├─ 1.3 环境依赖预检 ⚠️ 阻断型门禁
  │     └─ 读技术栈表 → 列依赖清单 → 逐包验证 → 缺失 npm install 一次性补齐
  ├─ 1.4 从 _template 复制原型骨架
  ├─ 1.5 定制模板（菜单/路由/页面映射）
  └─ 1.6 精读上游 4 个核心文件（PRD/页面清单/DESIGN.md/_同步校验记录）
      ↓
步骤二：核对清单输出 → 用户确认
  ├─ checklist.md（页面清单 + 每页模块/图表/操作点）
  └─ acceptance-checklist.md（从 PRD 交互表/异常表/闭环提取）
      ↓
步骤三：样板页面开发 → 用户目视确认
  ├─ 选覆盖面最广的页面（优先分析看板）
  ├─ 完整开发（功能/图表/交互/loading/空态/错误态）
  └─ 样板 = 后续所有页面的基准线
      ↓
步骤四：批量产出其余页面
  ├─ 页面类型→模板匹配（分析页/列表页/表单页/详情页/自定义）
  ├─ 逐页开发，每页对照对应线框图
  └─ 打通页面间导航（useHashPage）
      ↓
步骤五：验收与修复
  ├─ 5.0 流程检查点自检（12 项，3 阶段）
  ├─ 5.1 DESIGN.md 合规终检（26 项，4 层）
  ├─ 5.2 编译验收（check-app-ready.mjs → READY）
  ├─ 5.3 结构化业务验收（交互/异常/流程逐条核对）
  ├─ 5.4 客户端演示专项检查（地图/图表/编辑器/状态兜底）
  ├─ 5.5 输出验收报告
  └─ 5.6 闭环修复 → 全部 ✅
      ↓
步骤六：阶段收尾 → 更新 pipeline-status.yaml
```

## 质量保障体系

| 层级 | 机制 | 项数 | 触发时机 |
|:--:|------|:--:|---------|
| **GATE** | 流程检查点（阶段 A） | 2 项 | 写代码前，阻断型 |
| **CHECK** | 流程检查点（阶段 B） | 7 项 | 每页输出后，检查型 |
| **CHECK** | DESIGN.md 合规自检 | 26 项（T1 布局 5 + T2 Token 10 + T3 组件 7 + T4 状态 4） | 每页 + 最终验收 |
| **WARN** | 流程检查点（阶段 C） | 3 项 | 读上游时，预警型 |
| **RULE** | 布局硬约束 | 9 条 | 所有页面强制 |

## 布局硬约束（9 条）

| # | 规则 |
|:--:|------|
| 1 | 外层 Layout `height: '100vh'`（非 minHeight） |
| 2 | 内层 Layout `height: calc(100vh - 56px)` |
| 3 | Content `overflow: 'auto'`（内容独立滚动） |
| 4 | Sider `overflow: 'auto'` + `height: '100%'` |
| 5 | 分析看板 padding: 12px, gutter: [8,8], gap: 8px |
| 6 | 面包屑唯一承载页面标题（内容区不重复大标题） |
| 7 | 筛选栏: flex + space-between（左侧筛选，右侧操作，时间用 Segmented） |
| 8 | 单子菜单组 → 平级菜单项（无需展开） |
| 9 | 内容少的卡片 `height: '100%'`（不压缩内容多的卡片） |

## 组件模式

| 模式 | 结构 | 用途 |
|------|------|------|
| **StatCard** | `Card + Statistic(26px/600) + 迷你 ECharts(36px) + 趋势箭头` | KPI 指标卡 |
| **ChartCard** | `Card + title + 图表(≥200px)` | 图表容器卡片 |
| **筛选栏** | `flex + space-between`，左侧 Select/DatePicker/Segmented，右侧操作按钮 | 看板筛选 |
| **卡片等高** | 内容少的卡片 `height: '100%'` 撑满 Col | 同行卡片对齐 |
| **异常卡片** | flex 列布局 + 底部链接沉底 | 异常/告警列表卡片 |
