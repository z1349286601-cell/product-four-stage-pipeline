# Ant Design Pro 设计规范分析

> 来源：Ant Design 官方设计规范 + Ant Design Pro v2/v5 主题文档
> 目标：提炼为港口工业后台 DESIGN.md 的设计基底

---

## 1. 页面布局

### 经典三区布局
```
┌──────┬────────────────────────────────────────┐
│      │  Header（固顶，48px）                    │
│      ├────────────────────────────────────────┤
│Side  │                                         │
│bar   │  Content（主内容区，#f0f2f5 背景）        │
│暗色   │  ├── PageContainer（页头+操作）          │
│220px │  ├── Card / Table / Chart               │
│      │  └── Footer                             │
└──────┴────────────────────────────────────────┘
```

### 关键参数

| 元素 | 规范 |
|------|------|
| 侧边栏宽度 | 220px（暗色背景 `#001529`），可折叠至 80px |
| 头部高度 | 48px |
| 内容区背景 | `#f0f2f5` |
| 内容区内边距 | 24px |
| 栅格系统 | 24 栅格，Gutter 16px |
| 设计画板 | 1440px 基准，向下适配 1366 |

## 2. 配色体系

### 品牌色

| Token | 值 | 用途 |
|-------|-----|------|
| `primary-color` | `#1890FF` | 主按钮、链接、选中态 |
| `primary-hover` | `#40A9FF` | 悬停态 |
| `primary-active` | `#096DD9` | 点击态 |

### 功能色

| 功能 | 颜色 | 用途 |
|------|------|------|
| 成功 | `#52C41A` | 成功状态、正常标签 |
| 警告 | `#FAAD14` | 警告状态 |
| 错误 | `#FF4D4F` | 错误状态、删除按钮 |
| 信息 | `#1890FF` | 信息提示 |
| 链接 | `#1890FF` | 锚文本链接 |

### 中性色（10 阶灰）

| 色阶 | 浅色模式用途 |
|------|------------|
| `#000000` | — |
| `#000000E0` (87%) | 标题/一级文本 |
| `#000000A6` (65%) | 二级文本 |
| `#00000040` (25%) | 禁用文本 |
| `#D9D9D9` | 一级边框 |
| `#F0F0F0` | 背景色 |
| `#F5F5F5` | 浅灰背景 |
| `#FAFAFA` | 更浅背景 |

### 特殊背景

| Token | 值 | 用途 |
|-------|-----|------|

## 3. 字体体系

### 字体家族

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto,
  'Helvetica Neue', Arial, 'Noto Sans', sans-serif,
  'PingFang SC', 'Microsoft YaHei',  /* 中文优先 */
  'Apple Color Emoji', 'Segoe UI Emoji';
```

### 字阶与行高

| 层级 | 字号 | 行高 | 字重 | 用途 |
|------|:--:|:--:|:--:|------|
| H1 | 38px | 1.23 | 600 | 页面大标题（极少用） |
| H2 | 30px | 1.35 | 600 | 页面标题 |
| H3 | 24px | 1.35 | 600 | 模块标题 |
| H4 | 20px | 1.4 | 600 | 卡片标题 |
| H5 | 16px | 1.5 | 600 | 段落标题 |
| **Body** | **14px** | 1.57 (22px) | 400 | **正文标准** |
| Caption | 12px | 1.67 (20px) | 400 | 辅助说明文字 |
| Code | 13px | 1.57 | 400 | 代码/数字展示 |

**原则**：字阶控制在 3-5 种，多数后台页面用 14px 正文 + 16px 标题 + 12px 辅助。

## 4. 间距体系

### 基础单位：8px

```
间距 = 8 + 8 × n （n ≥ 0）
```

| Token | 值 | 用途 |
|------|:--:|------|
| xxs | 4px | 极小间距（图标与文字） |
| xs | 8px | 紧凑间距 |
| sm | 12px | 小间距（标签之间） |
| md | 16px | 常规内边距（Card/Form 内边距） |
| lg | 24px | 内容区间距（Content padding、模块间） |
| xl | 32px | 大模块间距 |
| xxl | 48px | 页面级间距 |

### CRUD 页面标准内边距

| 区域 | 内边距 |
|------|:--:|
| PageContainer | 24px |
| Card body | 16px（默认 24px，Ant Pro 推荐 16px） |
| Form item 间距 | 16px |
| Table 单元格 | 12px 8px（上下 12px，左右 8px） |
| Tabs 内嵌表格 | 0px（Tabs 自带间距） |

## 5. 表格规范（ProTable）

### 列配置

| 列类型 | 规范 |
|--------|------|
| 序号列 | 第一列，宽 60px，居中 |
| 主键/标题列 | 宽 ≥ 120px，蓝色链接 `#1890FF`，点击打开详情 |
| 状态列 | Badge/Tag 组件，彩色标签 |
| 操作列 | 固定右侧，宽 ≥ 120px，最多 3 个按钮 + 更多下拉 |
| 数字列 | 右对齐，开启 `sorter: true` |
| 日期列 | 居中，开启 `sorter: true` |

### 查询区域

| 属性 | 规范 |
|------|------|
| Label 宽度 | 88px（统一宽度，左对齐） |
| 栅格占位 | `span: 8`（每行 3 个筛选条件） |
| 默认显示 | 2 行，超出折叠，展开按钮"展开" |

### 表格交互

| 交互 | 规范 |
|------|------|
| Loading | 所有查询和操作必须有 loading |
| 空状态 | Empty 组件 + "暂无数据" |
| 错误状态 | Result 组件 + 重试按钮 |
| 批量操作 | FooterToolbar 悬浮条 |
| 分页 | 默认每页 20 条，支持切换 10/20/50/100 |

## 6. 表单规范（ModalForm / ProForm）

### 弹窗表单

| 表单项数量 | 布局 | 弹窗宽度 |
|-----------|------|:--:|
| ≤ 6 项 | 单列 | 400px |
| 6-12 项 | 两列 `grid colProps={{ span: 12 }}` | 560px |
| > 12 项 | 两列 / 改用 Drawer | 800px |

### 表单项规范

| 元素 | 规范 |
|------|------|
| 标签对齐 | 左对齐（可读性好）或顶部对齐（填写快） |
| 必填标识 | 红色 `*` 前缀 |
| 占位符 | 用"请输入/请选择"开头 |
| 输入框宽度 | 统一 100%（随容器），非固定 px |
| 校验提示 | 失焦后即时校验，红色文字提示 |
| 提交按钮 | 右对齐，主按钮 + 取消 |

### 提交反馈

| 结果 | 反馈 |
|------|------|
| 成功 | `message.success('保存成功')` + 关闭弹窗 + 刷新列表 |
| 失败 | `message.error('原因描述')` + 弹窗不关闭 |

## 7. 按钮体系

### 按钮类型

| 类型 | 样式 | 使用场景 |
|------|------|---------|
| `primary` | 品牌色填充 + 白字 | 页面唯一主操作（新建/保存/提交） |
| `default` | 白底 + 边框 | 次要操作（取消/返回） |
| `text` | 无底色无边框 | 表格操作列（编辑/查看） |
| `link` | 蓝色文字 | 链接式操作（详情/跳转） |
| `danger` | 红色 | 删除/停用/高危操作 |

### 尺寸规范

| 尺寸 | 高度 | 场景 |
|:--:|:--:|------|
| large | 40px | 页面级主按钮 |
| middle | 32px | 表格操作、表单内按钮（默认） |
| small | 24px | 紧凑场景 |

### 使用原则

| 原则 | 说明 |
|------|------|
| 一主一辅 | 每组按钮最多 1 个 primary + 1 个 default |
| 危险确认 | 删除类操作必须二次确认（Popconfirm） |
| 操作反馈 | 异步操作必须有 loading 态 |
| 禁用态 | 不满足条件时按钮 disabled，不隐藏 |

## 8. 弹窗规范

### Modal（操作型弹窗）

| 属性 | 规范 |
|------|------|
| 最小宽度 | 400px |
| 遮罩 | 黑色半透明 + 点击遮罩不关闭（防止误操作丢失表单数据） |
| 标题 | 左对齐，如"新建模板" / "编辑卡片" |
| 内容区 | 表单/文字/图表，滚动区域 |
| 底部按钮 | 右对齐，取消(default) + 确认(primary) |
| 关闭方式 | 右上角 × + 底部取消按钮 |
| 动画 | 淡入淡出 300ms |

### Drawer（详情/侧边面板）

| 属性 | 规范 |
|------|------|
| 触发 | 点击表格蓝色链接 / 点击按钮 |
| 宽度 | 560px（标准） / 800px（宽详情） |
| 位置 | 右侧滑出 |
| 内容 | Descriptions 两列 / 表单 / 图片 |
| 底部 | 可选操作按钮（编辑/删除） |
| 遮罩 | 浅遮罩，点击关闭 |

### 确认弹窗（Popconfirm）

```
气泡形式，用于删除/停用确认
├── 图标：红色感叹号
├── 文字："确定删除吗？"
├── 补充说明（选填）
├── [取消] [确认(红色)]
```

## 九、综合设计原则

| 原则 | 体现 |
|------|------|
| **统一性** | 所有交互反馈方式一致（message/notification/modal）；色彩、字体、间距全局统一 |
| **反馈性** | 所有可操作元素有 hover/active/disabled 三态；异步操作有 loading；操作结果有明确反馈 |
| **容错性** | 删除二次确认；表单即时校验；未保存离开提醒 |
| **效率性** | 高频操作用 ProTable 快捷键；批量操作用 FooterToolbar；详情用 Drawer 避免页面跳转 |
| **克制性** | 字阶 ≤ 5 种；颜色 6:3:1 比例；一页最多一个 primary 按钮 |
