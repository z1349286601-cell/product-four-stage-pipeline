## 第三轮优化（2026-07-03）：集装箱理货项目实战 + STAGE4 模板化改造

### 触发背景

用户使用更新后的流程模板创建"集装箱理货系统分析平台"项目，全流程实操中发现 12 条问题，推动本轮优化。

### 12 条流程问题（详见项目 `_流程问题经验.md`）

| 阶段 | 问题数 | 典型问题 |
|:--:|:--:|------|
| Stage 3 | 3 | 线框图格式错误（.md ASCII 替代 .excalidraw）、跳过逐页确认、图表过度绘制 |
| Stage 4 | 9 | 路径误读未确认、依赖未预检、布局未锁定视口、DESIGN.md 合规未校验、视觉冗余、卡片等高方向反、筛选栏无层次、菜单层级冗余、缺少布局切换 |

### 本轮关键决策

| 决策 | 说明 |
|------|------|
| **STAGE4 模板化** | 创建 `make-project-template/`，从零手写 → 模板启动。原型骨架 `_template/` 包含 Glass 布局外壳 + Dashboard 样板 + 合规清单 |
| **质量保障体系** | 建立三层质保：12 项流程检查点（GATE/CHECK/WARN）+ 26 项 DESIGN.md 合规清单（T1~T4）+ 9 条布局硬约束 |
| **环境依赖预检** | Stage 4 启动时增加阻断型门禁：读技术栈表 → 逐包验证 → 缺失一次性 npm install |
| **路径校验 Gate** | 用户提供的路径若 Test-Path 失败，必须先向用户确认，不可自行推断原因 |
| **Stage 3 加固** | CLAUDE.md 4.4 节新增三条强制规范：格式强制（excalidraw）、图表占位规范（不模拟数据图形）、逐页确认强制 |
| **双模板体系** | `_project_template/`（流水线项目骨架）+ `make-project-template/`（Make 工程骨架），新项目从两个模板复制启动 |

### 放弃的方向

- **ProCard npm 包**：最新 2.10.0 只支持 antd 4/5，make 项目用 antd 6.5.0，无法安装 → 零依赖 StatCard/ChartCard 组件模板
- **ProLayout npm 包**：强依赖 react-router，与 useHashPage 架构冲突 → 手写 Glass 布局
- **暗侧栏 Pro 风格**：用户实际跑过后觉得丑 → 白侧栏 + Glass Morphism
- **可复用工程文件模板**：用户说后续再考虑
