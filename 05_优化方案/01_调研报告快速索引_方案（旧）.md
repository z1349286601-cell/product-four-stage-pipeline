# 优化方案：调研报告内置 PRD 快速索引

> 方案日期：2026-07-01
> 改动范围：competitive-research 输出模板 + prd-generator 步骤一消费规则
> 原则：不改两个技能的核心流程，只加衔接层

---

## 一、改动一：competitive-research 新增「快速索引」模板

### 改动位置

调研报告输出模板中，「第一章 调研目标确认单」之后、「第二章 行业概况」之前，插入「第零章：PRD 生成快速索引」。

该索引由 competitive-research 技能在生成调研报告时**自动生成**，不增加用户操作步骤。

### 索引模板

```
## 零、PRD 生成快速索引

> 本章由 competitive-research 技能自动生成，为下游 PRD 生成工具提供结构化导航。
> 不含决策建议，仅为调研报告原文的指针索引。

### 0.1 报告数据概览

| 维度 | 数据 |
|------|------|
| 调研竞品总数 | {从第三章自动统计} |
| 功能模块数 | {从第四章自动统计} |
| 功能点总数 | {从第四章自动统计} |
| 识别用户角色 | {从第五章用例图自动提取} |
| 核心业务流程 | {从第五章统计} |
| 涉及标准规范 | {从第九章统计} |

### 0.2 信息类型 → 调研报告对照表

| 信息类型 | 调研报告参考位置 | 关键数据摘要 |
|---------|-----------------|-------------|
| **行业背景与趋势** | 二、行业概况 | 市场趋势、产业链结构、主流玩家分类 |
| **用户角色画像** | 五.1 用例图 + 三.2 产品定位对比 | 用例图中的 actor 清单、竞品目标客户群 |
| **核心业务场景** | 五.2 活动图 + 五.3 时序图 | 主要业务流程的 PlantUML 源码 |
| **竞品功能全景** | 四、详细功能清单对比表 | {N} 竞品 × {N} 功能点的 ✅/❌/部分 对比矩阵 |
| **竞品能力差异** | 七、竞品对比总结 | 各竞品优势清单 + 短板清单 |
| **功能优先级参考** | 八、功能优先级建议 | P0/P1/P2 分级及其理由 |
| **流程建模素材** | 五、UML 图（全 {N} 张） | 用例图/活动图/时序图/状态机图 PlantUML 源码 |
| **行业合规标准** | 九、标准规范体系 | 国内/国际标准编号、名称、适用场景 |
| **通用术语定义** | 十一.1 术语表 | {N} 个行业核心术语及定义 |
| **系统集成方向** | 十一.3 待深入研究项 | 待验证的集成/对接方向 |
| **差异化机会** | 七.3 差异化机会 | 竞品未充分覆盖的方向列表 |

### 0.3 可复用的结构化数据

| 数据类型 | 位置 | 格式 | 说明 |
|---------|------|------|------|
| 功能对比矩阵 | 四、功能清单对比表 | 表格（✅/❌/部分） | 可直接作为功能参考基准 |
| PlantUML 源码 | 五、UML 图 | PlantUML 代码块 | 可直接渲染或编辑 |
| 标准规范清单 | 九、标准规范体系 | 表格（编号/名称/内容/场景） | 可直接引用为合规需求依据 |
| 竞品优劣势列表 | 七、竞品对比总结 | 结构化列表 | 可作为模块决策参考信息 |
| 术语定义表 | 十一.1 术语表 | 表格（术语/英文/定义） | 可直接引用 |

### 0.4 使用说明

**推荐消费顺序**：
1. 先读本章索引（0.1-0.3），了解报告全貌
2. 按所需信息类型，跳转到调研报告对应章节读取详情
3. 功能对比数据直接引用第四章表格，不再二次提取
4. UML 流程图直接复用第五章 PlantUML 源码

**⚠️ 禁止直接从调研报告推断的内容**：
以下内容调研报告只提供参考信息，最终决策必须与用户对话确认，禁止 AI 自行推断或直接采纳：

| 禁止推断的内容 | 原因 | 正确做法 |
|--------------|------|---------|
| **产品定位** | 这是业务决策，调研只提供行业背景 | 与用户对话确认目标行业/客群/价值主张 |
| **目标用户选择** | 调研列举的是竞品的用户群，不是你产品的 | 与用户确认本项目实际面向的用户角色 |
| **功能取舍（做什么/不做什么）** | 调研告诉你竞品做了什么，不是你应该做什么 | 基于用户需求对话确认功能范围 |
| **模块优先级决策** | P0/P1/P2 是基于竞品共性分析的建议，不是你的产品路线图 | 与用户确认实际优先级 |
| **技术架构选型** | 调研不覆盖技术决策 | 与用户及技术团队确认 |
| **页面/交互设计方向** | 调研报告不含设计决策 | 在 PRD 阶段与用户逐步确认 |

**索引维护说明**：
- 本索引由 competitive-research 技能在生成调研报告时自动生成
- 自动统计 0.1 中的数据维度
- 自动映射 0.2 中的信息类型与章节对应关系
- 用户无需手动维护
```

## 二、改动二：prd-generator 步骤一增加索引消费规则

### 改动位置

`prd-generator` 技能 `步骤一：资料输入与解析` → `1.2 资料类型识别与提炼`，在现有资料类型识别表中新增一行：

### 新增行

```markdown
| **调研报告（含快速索引）** | 优先读取报告头部「PRD 生成快速索引」章节（第零章），
通过 0.2 信息类型对照表定位所需数据在报告中的位置，按需跳转读取详情 |
```

### 同时，在步骤一的「1.3 信息确认」后增加一条提示规则

```
若输入的调研报告包含「PRD 生成快速索引」章节：
- 通过索引定位数据，而非通读全文
- 功能对比矩阵直接引用，不二次提取
- UML 流程图直接复用 PlantUML 源码
- 严格遵守索引 0.4 中的「禁止推断」清单，涉及决策项必须与用户对话确认
```

---

## 三、改动影响评估

| 评估维度 | 影响 |
|---------|------|
| competitive-research 核心流程 | 无影响，仅在输出模板增加一个自动生成的章节 |
| prd-generator 核心流程 | 无影响，仅在步骤一增加一条资料类型的识别规则 |
| 用户操作 | 无变化，索引自动生成，用户无额外操作 |
| 向后兼容 | 完全兼容，旧版调研报告无索引时，PRD 生成器按原逻辑工作 |
| 港口行业适配 | 索引模板通用，不绑定特定行业 |

---

## 四、效果对比

| 维度 | 改前 | 改后 |
|------|------|------|
| PRD 生成器对调研报告的处理 | 通读 555 行全文，随机采样 | 先读 ~40 行索引，按图索骥精准跳转 |
| 功能对比表覆盖率 | 易漏模块（采样不全） | 0.3 告知完整模块数和位置，逐一覆盖 |
| UML 图利用率 | 低（容易被忽略） | 0.3 标注可直接复用 |
| AI 越界决策 | 可能从调研报告直接推断产品定位 | 0.4 禁止清单约束，强制与用户确认 |
| 耦合度 | 索引直接绑定 PRD 章节编号 | 信息类型通用，不依赖特定 PRD 模板 |
