总目标
全程自治执行，依据上传的《需求规格说明书》《UI 规范说明》，产出一套可本地直接运行的 Vue3 + Vite + TypeScript 可交互 HTML 页面原型。所有页面必须 1:1 对齐需求说明书「详细功能页面设计」章节的功能、字段、模块、交互逻辑，UI 严格匹配规范，图表完整可用，最终通过五级验收后自动结束任务，不达标则循环修复直至全量通过。
一、最高优先级：需求对齐与长任务专注力规则
本章节优先级高于所有技术、样式要求，违反视为验收不通过
唯一业务基准：所有页面、功能、字段、交互，必须可追溯至《需求规格说明书》中「详细功能页面设计」章节原文。开发前先提取该章节全部页面清单、每个页面的模块 / 字段 / 操作点，形成开发核对清单，严格按清单逐项落地，禁止遗漏。
严格范围管控：禁止主动新增需求文档未提及的功能、页面、动效、创意优化；不得自行简化、合并需求中明确列出的功能模块。
分阶段递进执行：按下方阶段顺序推进，前一阶段自检通过后方可进入下一阶段；每完成一个阶段输出一行「阶段完成摘要」，避免一次性发散导致偏离需求。
歧义处理规则：若文档描述存在歧义，优先按工业后台通用实现逻辑落地，同时在验收报告中标注歧义点与处理方案，禁止擅自变更业务定义。
中途不跑偏约束：全程只围绕本原型开发目标执行，不引申技术方案讨论、不做额外架构设计、不生成无关文档。
二、技术栈强制约束（不可变更）
核心框架：Vue 3 + Vite + TypeScript，使用 ESM 模块规范
UI 组件库：Ant Design Vue 4.x，全局引入官方 reset 样式，组件默认配置统一在 app.use 处全局配置
图表方案：vue-echarts + ECharts 5.x，所有图表容器绑定窗口 resize 事件自动重绘，tooltip、图例、维度切换齐全
图标库：@ant-design/icons-vue，统一按需引入
样式规范：单文件组件使用 scoped CSS；全局色值、字号、间距、圆角、阴影全部提取至 constants/ui.ts 常量文件，页面内禁止硬编码色号、固定像素尺寸
路由方案：vue-router 4.x，单页面路由结构，路由与「详细功能页面设计」章节页面一一对应，菜单与路由联动
数据层：全部使用 Mock 模拟数据，封装在 api/ 目录下，页面仅调用方法不写死数据；所有操作附带 loading 状态、空数据态、请求失败兜底提示
地图组件：涉及地图的页面统一使用高德地图 JS API，密钥固定为 bd77af8b7b13b4c563f7fd06fe4a68c3，按需懒加载，不全局冗余引入
目录结构强制分层：views / components / api / constants / router / utils / assets，公共组件与页面局部组件分层存放
三、分阶段执行流程（保障长任务有序推进）
阶段 1：工程初始化与基建
生成完整 package.json，锁定依赖版本，确保无冲突
搭建目录结构、路由骨架、全局样式入口、UI 常量文件、请求 Mock 封装
执行 npm install 安装依赖，确认环境无报错
阶段 2：公共组件与图表封装
封装通用表格、表单弹窗、筛选栏、空状态、加载态等全局公共组件
封装 ECharts 通用容器，统一适配逻辑、默认样式、主题配置
校验公共组件可正常引用，无类型报错
阶段 3：核心页面逐页开发
严格按照「详细功能页面设计」章节的页面顺序，逐页开发
每页完成后即时自检：模块完整性、字段齐全度、操作入口匹配度
涉及地图、图表的页面同步完成 Mock 数据渲染
阶段 4：全量交互联调
打通页面间跳转、弹窗开闭、表单校验、搜索筛选、分页排序、批量操作等全部交互
补全所有异常态、空态、加载态、操作反馈
全局样式统一巡检，确保所有页面符合 UI 规范
阶段 5：全量验收与闭环修复
执行五级验收校验，生成验收自检报告
对未通过项自动定位修复，修复后重新执行对应校验
循环迭代直至所有校验项 100% 通过，goal 任务完成
四、五级强制验收体系（goal 完成唯一判定标准）
所有校验项必须逐条输出「通过 / 未通过」状态，未通过项附带问题定位与修复记录；全量通过后方可判定任务完成。
校验 0：业务一致性校验（核心对标「详细功能页面设计」）
页面数量、页面名称、菜单结构与需求文档完全一致，无缺页、无多余页面
每个页面的功能模块、展示字段、操作按钮、筛选条件，与文档描述一一对应
业务逻辑、状态流转、权限展示逻辑符合文档定义
输出《页面功能匹配对照表》，逐页标注匹配情况
校验 1：编译运行校验
npm run build 执行成功，0 error，0 warning（含 TS 类型警告）
npm run dev 启动后控制台无报错，页面可正常访问无白屏
路由跳转正常，无 404、无组件渲染失败
依赖版本兼容，无冲突、无废弃 API 警告
校验 2：UI 还原度校验
主色、辅助色、文字色、边框色、背景色与 UI 规范完全一致
字号、行高、内边距、外边距、圆角、阴影、按钮尺寸、表格行高严格对齐规范
hover /active/disabled 三种交互状态样式匹配 UI 文档
布局间距、对齐方式、栅格系统符合规范
校验 3：交互功能校验
搜索筛选：输入筛选、重置清空、多条件组合生效

状态反馈：加载中动画、空数据状态、操作成功 / 失败提示齐全
校验 4：代码规范校验

UI 常量统一存放，页面无硬编码色值、固定尺寸
组件命名、文件命名、目录结构符合前端规范
API 层与页面解耦，Mock 数据统一管理，业务逻辑有必要注释
